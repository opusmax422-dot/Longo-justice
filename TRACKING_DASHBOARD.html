How to Identify Suspicious Activity

Unknown IP Addresses

Look for IP addresses that don't match your known devices or locations:

  • IP addresses from government ranges (see IP Analysis section)
  • IPs from locations you've never visited
  • Multiple IPs accessing the site in quick succession
  • IPs that appear repeatedly over days/weeks

Access to Hidden Pages

Automated User-Agents

Look for user-agents indicating automated scraping/surveillance:

  • Government monitoring tools (e.g., "GovernmentBot", "LEMonitor")
  • Headless browsers (e.g., "HeadlessChrome", "PhantomJS")
  • Web scrapers (e.g., "curl", "wget", "Python-requests")
  • Unusual or outdated browser versions

Clone Domain Detection Alerts

The clone-detector.js script triggers when the site is served from an unauthorized domain:

  • Alert type: CLONE_DETECTED
  • Check the "unauthorizedDomain" field to see where it was hosted
  • This proves the site is being mirrored/proxied without authorization

Timeline Reconstruction

Cross-Reference Activity

Step 1: Note Your Activity

Keep a log of when YOU access the evidence sites. Note the time, device, and network.

Step 2: Check Webhook Timestamps

Compare webhook request timestamps to your activity log. Any requests during times you were NOT accessing the site = unauthorized access.

Step 3: Identify Access When You Were Offline

CRITICAL: Any webhook requests when you were asleep, offline, or away from devices = proof of unauthorized surveillance.
